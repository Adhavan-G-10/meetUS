## 📊 Technical Architecture

### **Signaling Flow (Socket.io)**
```
User A                    Server                   User B
   │                         │                        │
   ├─── webrtc-offer ───────>│─── webrtc-offer ──────>│
   │                         │                        │
   │<─── webrtc-answer ──────│<─── webrtc-answer ─────┤
   │                         │                        │
   ├── webrtc-ice-candidate │── webrtc-ice-candidate >│
   │                         │                        │
   │< webrtc-ice-candidate ──│< webrtc-ice-candidate ─┤
```

### **Media Flow (Peer-to-Peer)**
```
User A Video/Audio Stream ←→ User B Video/Audio Stream
  (via RTCPeerConnection)
```

### **Connection States**
```
CONNECTING → (Exchange SDP & ICE) → CONNECTED → STREAMING
```
